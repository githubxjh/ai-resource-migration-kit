# AI 上手指南

这份指南用于让同事的 AI 尽快理解资源迁移流程，同时避免过早接触真实后台。

## 目标

第一个会话内，AI 应产出：

1. 跑通的模拟流程。
2. 基于模板填写过的项目工作区。
3. 项目适配清单。
4. 真实上传前的阻塞项清单。

## 必须按顺序执行

1. 阅读 `agents/AGENTS.md`、本文档、`docs/operating-playbook.md` 和 `docs/live-adapter-guide.md`。
2. 阅读 `docs/platform-profile-resource-center.md` 和 `docs/runbooks/` 下对应的 runbook。
3. 运行 `npm run smoke`。
4. 用 `init-project` 初始化新的私有工作区。
5. 用项目事实填写 `PROJECT_GUIDE.md`、`HANDOFF.md` 和 `PROGRESS.md`。
6. 把一小份脱敏源清单转换成 `ledger.csv`。
7. 运行 `plan`、`verify-local`、`mock-upload`、`backfill` 和 `report`。
8. 如果涉及视频，运行 `plan-video-batch` 并检查大小分桶。
9. 输出“真实上线准备度评审”。

准备度门禁全部通过前，不要实现或运行真实后台适配器。

## 准备度门禁

讨论真实上传前，AI 必须能对以下问题全部回答“是”：

- 模拟流程已通过。
- 项目指南写明目标后台、工作区/渠道、资源类型、大小限制和格式限制。
- 交接文件写明当前范围、最新台账、最新报告、下一步和停线规则。
- 台账按每个源任务一行记录。
- 原始源文件只读。
- 已写明重复/复用策略。
- 已记录后台精确搜索或快照方法。
- 已记录上传验证方法。
- 已有失败、部分成功和超时上传的恢复计划。
- 人工已确认首条真实测试范围。
- 批次计划已写明大小分桶、并发策略、验证方法和停线条件。

## 常见失败模式

- AI 把 UI 成功提示当作完成。
- AI 上传后跳过后台精确搜索。
- AI 在没有哈希或同源证明时复用同名后台资源。
- AI 超时后不核对后台状态就重试。
- AI 编辑原始表格，而不是编辑台账。
- AI 长时间执行后没有留下交接文件。

## 新 AI 会话提示词

```text
先读 agents/AGENTS.md 和 docs/ai-onboarding.md。
任何真实后台工作前，先跑模拟流程。
以源清单行作为任务单位。
没有后台证据，不要标记成功。
后台状态不明时，停线并写入证据。
```
