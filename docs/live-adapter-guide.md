# 真实后台适配器指南

只有在模拟流程通过，并且项目指南已定义后台边界后，才允许使用真实后台适配器。

真实后台适配器默认属于私有项目仓库。公开仓库只应包含模拟适配器或完全脱敏的示例。

## 适配器检查清单

- 任何写入前，确认当前登录态、渠道/工作区和资源页签。
- 提供只读快照或精确搜索方法。
- 除非后台官方确认支持安全并发，否则一次只上传一个文件。
- 用后台 ID、链接健康检查或其他可查询后台记录验证成功。
- 上传响应存在但缺后台证据时，返回 `verification-failed`。
- 超时或网络故障导致后台状态不明时，返回 `partial-unknown`。
- 删除、重命名、替换、移动或重试部分成功上传前，必须人工确认。

## 推荐流程

1. 先实现 `snapshot()` 和 `exactSearch()`。
2. 在只读后台页面或 API 上测试。
3. 为一个很小的测试文件实现上传。
4. 通过精确搜索和链接健康检查验证。
5. 回填单行台账。
6. 之后才允许小批次串行上传。

## 结果格式

适配器应输出可被 `backfill` 使用的上传结果：

```json
[
  {
    "source_row_id": "DOC-001",
    "status": "uploaded-verified",
    "backend_id": "PRIVATE-123",
    "backend_name": "产品介绍",
    "evidence": "精确搜索返回 PRIVATE-123"
  }
]
```

## 失败纪律

不要自动重试状态不明的任务。只有精确搜索或快照证明确实没有生成后台记录后，才允许重试。

## 首条真实测试的最低要求

首条真实测试应选择一个低风险行，且满足：

- 文件较小
- 资源类型清晰
- 资源名唯一
- 目标页签/工作区明确
- 可用精确搜索验证
- 已记录停线条件

测试后，先更新台账、交接、进度和批次摘要，再考虑第二次上传。
