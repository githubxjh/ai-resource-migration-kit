# 视频上传 Runbook

资源中心类后台的视频资源使用这份 runbook。

## 大小分桶

| 分桶 | 策略 |
|---|---|
| `<50 MB` | 精确预搜索后，可作为普通小批次候选。 |
| `50-500 MB` | 优先使用 file input 或后台支持的上传路径；避免对较大文件使用 base64/CDP 路径。 |
| `500 MB-2 GB` | 单条或极小串行批次；判断失败前允许页面安静等待数分钟。 |
| `near 2 GB` | 放在队列最后；只按单条处理；保持页面和网络稳定。 |
| `>2 GB` | 不按原样上传；标记为压缩、拆分或平台侧决策。 |

## 名称规则

- 默认保留原始资源名。
- 约 64 字符的名称先视为风险候选。
- 短名是异常兜底路径，不是默认策略。
- 如果需要短名 staged copy，记录原名、短名、staged 路径、后台实际名称、后台 ID 和原因。

## 上传路径建议

- 中大型视频避免使用浏览器/CDP base64 上传路径。
- 优先使用 file input，或能流式传到对象存储的私有适配器。
- 对象存储成功不等于完成；必须创建后台资源记录并通过精确搜索验证。
- 遇到 token 过期或超时，重试某一行前必须先刷新或快照后台状态。

## 停线条件

遇到以下情况立即停线：

- 上传弹窗残留上一次尝试的文件。
- 资源名超过已确认的平台限制。
- runtime/base64 评估超时。
- token 过期或上传返回未知状态。
- 看似成功后，后台精确搜索没有返回 ID。
- 渠道/工作区/页签未确认。

## 快速完成策略

当目标是尽快完成时：

1. 先上传安全的小视频。
2. 再处理 50-500 MB 视频。
3. 将 500 MB-2 GB 视频排成单条批次。
4. 将 `>2 GB`、已知源文件问题、等待重复复用确认和短名候选放入后处理清单。
