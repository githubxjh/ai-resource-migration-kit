# 文档上传 Runbook

文件、PDF、Office 文档、压缩包和其他资料类资源使用这份 runbook。

## 本地检查

- 确认文件存在且不是 0 字节。
- 检查文件签名，不要只看扩展名。
- PDF 文件应以 `%PDF` 开头。
- Office 文档通常使用 ZIP 类文件签名。
- 只有后台允许时，才把压缩包当作文档资源处理。

## 长 basename 风险

部分资源中心在 basename 过长时，可能能创建后台记录，但生成的对象链接是坏的。长文档 basename 应视为链接风险候选。

公开版默认规则：

- 如果 basename 长度超过项目阈值，先暂停并确认 staged copy/短名策略。
- 上传后先做链接健康检查，再标记完成。

## 完成口径

文档只有在以下条件满足时才算完成：

- 后台精确搜索返回 ID 或等价稳定记录。
- 如果平台暴露链接，链接健康检查通过。

## 大小限制和失败处理

- 如果后台因大小拒绝文档，标记 `blocked-size-limit`。
- 可考虑压缩、拆分 PDF、扫描替代件或平台侧处理。
- 如果文档对象存储已传输但资源记录未创建，任何重试前都要先精确搜索。

## 面向客户的名称

- 默认保留原始名称。
- 只有平台拒绝原名，或链接健康检查证明原路径不安全时，才使用短名。
- 短名应便于人识别，不要使用内部行号。
