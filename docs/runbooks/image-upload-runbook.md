# 图片上传 Runbook

资源中心类后台的图片资源使用这份 runbook。

## 本地检查

- 信任扩展名前，先检查真实文件头。
- 拒绝 0 字节文件。
- 项目确认前，先把约 5 MB 当作图片默认限制。
- 格式不一致时，先离线修复再上传。

## 上传组件证据

对于 Element UI 或类似上传组件：

- 框架接收文件后，原生 `input.files` 可能被清空。
- 不要只凭 `input.files.length === 0` 判断选择失败。
- 可用时，检查框架上传状态或可见已选文件列表。
- 选择下一个文件前，清理上传组件残留状态。

## 重复和自动后缀行为

- 同一资源名但格式不同，可能导致后台自动加后缀。
- `(1)` 这类自动后缀不一定是失败。
- 必须记录实际后台名称和后台 ID。

## 压缩和优先级

- 尽量保留图片质量。
- 只有源文件和格式检查证明超限后，才考虑压缩。
- 如果目标是尽快完成，先上传已确认安全的图片，把压缩/格式修复列入明确后处理队列。

## 完成口径

图片任务只有在后台精确搜索返回创建的资源 ID 或等价稳定证据后才算完成。
