# 资源迁移

使用本技能执行资源迁移项目，重点是证据充分、状态可恢复。

## 首次读取

开始任何项目工作前，先读取：

1. `HANDOFF.md`
2. `PROJECT_GUIDE.md`
3. `PROGRESS.md`
4. `ledger.csv`
5. 如存在后台适配器文档，也要读取

如果这些文件不存在，先用仓库模板初始化，再开始规划。

真实后台工作前，读取 `docs/ai-onboarding.md` 并通过模拟流程门禁。

## 核心规则

- 以源清单行作为任务单位。
- 永远不要修改原始源表格。
- 上传成功必须有后台 ID 或等价可查询证据。
- 不要把 UI 成功提示当作完成。
- 没有哈希、同源证明或明确人工确认时，不要复用后台资源。
- 除非后台适配器证明安全并发，否则上传必须串行。
- 渠道、登录态、页签、重复状态、超时、部分上传或验证缺失不清楚时，必须停线。
- 暂停前更新交接、进度、台账和报告。

## 工作流

1. 从源清单建立或刷新台账。
2. 检查本地文件：是否存在、大小、扩展名、类型签名，必要时计算哈希。
3. 上传前获取只读后台快照或执行精确搜索。
4. 只上传符合条件的行，并使用小批次验证。
5. 用后台 ID 验证每次上传。
6. 将结果回填台账。
7. 导出内部报告和客户核查报告。
