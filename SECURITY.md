# 安全策略

这个仓库按公开分享设计。不要提交真实迁移数据。

## 严禁提交

- 客户名称或源表格。
- 真实后台 URL、资源 ID、cookie、token、请求头或后台快照。
- 原始文件或客户提供的素材。
- 真实后台的上传日志。
- 本地浏览器 profile 或自动化状态。
- 包含平台专用接口、路径或鉴权细节的私有后台适配器。

## 推荐的私有目录结构

公开框架代码和私有项目工作区要分开：

```text
ai-resource-migration-kit/          公开框架
private-migration-projects/         私有项目工作区
  customer-a/
    adapters/
    source/
    workspace/
```

如果某个私有适配器值得复用，必须先脱敏，把真实接口替换成占位符，再考虑提交到公开仓库。

## 发现泄漏时

如果敏感信息被误提交，先轮换受影响的凭据，再从 Git 历史中清理，最后才允许公开发布。
